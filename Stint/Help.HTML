Starting an Event

Type a title in the Action field and press Enter. Optionally press Tab first to add a category — entering a category locks you into creating a new event regardless of any active event match.

Both the Action and Category fields offer autocomplete based on your history. Use ↓ to move into the suggestion list, Enter to select, and Escape to return to the field. Tab on a highlighted suggestion selects it and moves to the next field.

Retroactive Start

If you forgot to start an event, prefix your entry with a time using @HH:mm: 

  @09:30 Morning standup

The event will be recorded as having started at that time. The time prefix is stripped from the saved title. Times must be in the past — future times fall through to a normal event start.

Stopping an Event

There are two ways to stop a running event:

  • Type #n — e.g. #2 or #02 to select by row index. Press Enter to stop.
  • Double-click a row in the active events list to open the edit form, then click Stop.

Editing an Event

The edit form lets you change the title, category, start and stop times, stop a running event, or delete it entirely.

  • Type !n — e.g. !2 to open the edit form for active event #2.
  • Double-click a row in the active events list.
  • Click the edit link in any history report.

Undoing an Accidental Stop

Press Ctrl+Z immediately after stopping an event to restore it. The original start time is preserved — no time is lost.

Auto-Stop

Events are automatically stopped after a configurable maximum duration (default: 8 hours). This is intended to catch events left running overnight. You will receive a notification at the 50% mark as a heads-up, and another when the event is auto-stopped.

Configure or disable this in config.txt: 

  
    max_duration_minutes=480
    max_duration_minutes=off

Reports

Type a report command in the Action field and press Enter. Reports open in your default browser and are saved to the configured reports folder.

Command Description
?today Totals for today — by event and by category
?yesterday Totals for yesterday
?week Totals for the current working week (Monday–today)
?month Totals for the current calendar month
?history Full log of every event ever recorded
?open Opens the reports folder in Windows Explorer
?help Opens this help page

URL Protocol — stint://

Stint registers a custom URL scheme that allows external applications, scripts, and browser links to interact with the app. The running instance receives the URL and acts on it immediately — no second instance is launched.

URLs can be triggered from a browser address bar, Win+R, shell scripts, batch files, or any application that can open a URL.

URL Description
stint://show Brings the Stint popup to the foreground
stint://start?title=Meeting Starts a new event with the given title
stint://start?title=Meeting&category=DEV Starts a new event with title and category
stint://start?title=Meeting&category=DEV&time=09:30 Starts a new event with a retroactive start time
stint://start?title=Meeting&guid=your-guid Starts a new event with an external reference ID (silent ignore if duplicate)
stint://stop?id=42 Stops the event with the given database ID
stint://stop?guid=your-guid Stops the event with the given external reference ID
stint://edit?id=42 Opens the edit form for the given event (coming soon)
stint://report?type=today Generates and opens a report — types: today yesterday week month history

URL Encoding

Spaces and special characters in parameter values should be URL encoded. Spaces can be represented as %20 or +. 

  stint://start?title=Daily%20Standup&category=MEETINGS

From a Batch File or PowerShell

  
    :: Batch
    start stint://start?title=Lunch+Break

    # PowerShell
    Start-Process "stint://start?title=Lunch+Break&category=PERSONAL"

Configuration

A config.txt file is created alongside the executable on first run. Delete it to regenerate defaults.

Setting Default Description
hotkey Alt+Shift+T Global hotkey to show/hide the app. Supports Alt, Shift, Ctrl modifiers.
max_duration_minutes 480 Auto-stop threshold in minutes. Set to off to disable.
report_path .\Reports Folder where generated reports are saved.

Data

All event data is stored in a SQLite database at:

  %LOCALAPPDATA%\Stint\stint.db

This file persists across restarts and can be queried directly with any SQLite tool for custom reporting.
